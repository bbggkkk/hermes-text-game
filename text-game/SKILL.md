# Hermes Text Game Skill (Discord Formatter Edition)

에이전트가 **게임 마스터(GM)**가 되어 사용자가 원하는 텍스트 게임을 운영하되, 출력의 형태(UI)는 반드시 **디스코드 포맷팅 엔진**을 통해서만 사용자에게 보여줍니다.

이 방식은 장르적 하드코딩(규칙, 턴 처리, 데미지 계산 등)은 모두 에이전트의 자율적 판단에 맡기되, **화면에 어떻게 렌더링될지(포맷팅)만 스크립트로 강제**하는 구조입니다.

---

## 작동 방식

### 1. 게임 진행
에이전트는 사용자와 대화하며 게임을 진행합니다. 매 턴마다 장르와 현재 상황에 맞게 스토리, 상태 변수, 선택지 등을 결정합니다.

### 2. 출력 포맷팅 강제 (필수)
에이전트는 사용자에게 게임 화면을 렌더링해서 보여줄 때, **절대 스스로 마크다운을 직접 작성하지 않습니다.**
반드시 자신이 구상한 장면 데이터를 **JSON**으로 구성한 뒤, 아래 스크립트를 호출해야 합니다.

```bash
python3 text-game/scripts/discord_engine.py --data '<JSON_STRING>'
```

### JSON 구조 명세
`--data` 인자에 넘길 JSON 문자열은 다음 형태여야 합니다:

```json
{
  "title": "현재 장소나 상황의 제목 (예: 어두운 동굴)",
  "description": "상황에 대한 묘사 (2~4문장)",
  "status": {
    "HP": "100/100",
    "소지품": "횃불, 녹슨 검",
    "상태": "독에 걸림"
  },
  "events": [
    "플레이어가 문을 열었습니다.",
    "함정이 발동하여 10의 피해를 입었습니다."
  ],
  "choices": [
    {"key": "1", "text": "앞으로 나아간다."},
    {"key": "2", "text": "포션 마시기"}
  ],
  "game_over": false
}
```

- **status**: 해당 장르에 맞는 키-밸류 쌍을 자유롭게 넣으면 엔진이 `yaml` 코드 블록으로 변환해 줍니다.
- **game_over**: `true`일 경우 선택지 대신 게임 종료 메시지가 렌더링됩니다.

### 3. 결과 전달
스크립트가 출력한 `STDOUT` 내용(디스코드 포맷 마크다운)을 **그대로 복사하여 사용자에게 응답**합니다.

이 장치를 통해 에이전트는 장르에 구애받지 않는 자유로운 텍스트 게임을 운영하면서도, 시각적인 구조는 완벽하고 통일된 디스코드 UI로 고정할 수 있습니다.
